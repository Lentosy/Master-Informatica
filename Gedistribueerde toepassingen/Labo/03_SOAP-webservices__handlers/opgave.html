Aanpassen PlantService

In de labo gebruiken we twee types webservice handlers om de berichten naar de webservice PlantService uit het vorig labo te parsen en eventueel om te vormen. Lees sectie 6 uit de jax-ws gebruikersgids over webservice handlers. De volgende pagina legt uit hoe je handlers kan toevoegen in Netbeans.

LogicalHandler

Gebruik een LogicalHandler om te loggen hoeveel keer planten van een bepaalde kleur gevraagd worden. Voeg in Netbeans een LogicalHandler toe en configureer de webservice zodat de handler de berichten kan parsen. Welke bestanden worden er aangemaakt/aangepast? Haal in de handler de inhoud van het SOAP-bericht op en bekijk met de debugger wat het type is van het object dat de inhoud voorstelt. Zoek in de API op welke interfaces deze klasse implementeert en gebruik die om de kleur te bepalen. Test uit via de webinterface (zie ook Getting Started with JAX-WS Web Services). Dit kan enkel als je Glassfish als JE22-server gebruikt. Je kan de XSLT API en een ByteArrayOutputStream gebruiken om de inhoud van het bericht om te zetten naar tekst.

SOAPHandler

De plantservice verwacht dat de kleur van de planten in het Nederlands wordt ingegeven. Schrijf een SOAPHandler die toelaat om planten op kleur op te vragen in het Engels. Voeg in Netbeans een SOAPHandler toe en configureer de webservice zodat de handler de berichten kan parsen. Haal in de handler de inhoud van het SOAP-bericht op en bekijk opnieuw met de debugger wat het type is van het object dat de inhoud voorstelt. Zoek in de API op welke interfaces deze klasse implementeert en gebruik die om het SOAP-bericht van de client aan te passen zodat de service dit bericht toch kan afhandelen ( m.a.w. vertaal de kleur naar het Nederlands). Gebruik een XSLT, inlezen met een ClassLoader, om het bericht om te vormen. Test uit via de webinterface en met jouw clientapplicatie. Kan je bij een nieuwe plant de kleur ook in het Engels opgeven?
